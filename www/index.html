SvnQuery is a fast full text search engine for subversion repositories. It searches every file in every revision. Not only the content of a file is indexed, but also its complete metadata like path, author, comments and properties. A simple, Google-like web frontend is provided, but it is hoped that the query library and indexing mechanism can be reused in other frontends, e.g. a web service or a Visual Studio Plug-In.

SvnQuery is implemented in C# 3.0 and ASP.NET is used for the web frontend. The full text search is done by Lucene.Net which delivers an astonishing search speed.

The interface to subversion is isolated so that the svn command line client or something like SharpSvn can be used as an implementation. Porting to Java using the original implementation of Lucene should be easy.

Roadmap

The current version 0.7 is feature complete, well tested and working in a real life environment. The only missing feature is some kind of syntax highlighting for common languages (at least C, C++, C#, Java, Python, Xml). We are still searching for other open source projects that could do the job, but theiy are either not capable of highlighting big files (>30KB) or are not usable in an ASP.NET environment.
