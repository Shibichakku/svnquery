How to setup SvnWebQuery with IIS 6.0 on Windows Server 2003

It is assumed that IIS and the .NET Framework 3.5 has been installed. In this example the SvnWebQuery folder is located at C:\SvnWebQuery and the repository index is located at E:\RepositoryIndex.

  1. Startup the IIS Manager and create a virtual directory
  2. A wizard will help you ...
  3. Enter a name under which the virtual directory will be known by the wold. (This will be the address you have to enter inside a browser to navigate to the SvnWebQuery page)
  4. Choose the path to the SvnWebQuery directory from the distribution (e.g. C:\SvnWebQuery)
  5. Set the access permissions (note that you have to check the Run scripts to enable ASP.NET)
  6. The wizard has finished, but setup is not yet complete
  7. Open the property page of the newly created virtual directory (aka Web Site) SvnWebQeury
  8. On the "Virtual Directory" page click the "Create" button
  9. The newly created application should be now automatically named SvnWebQuery. Very that the execute permissions are set to "Scripts only"
  10. On the "Documents" page enable the default content page to be "query.aspx". This is the main page of our application. If you forget this step you will have difficulties to browse to the search page later.
  11. On the ASP.NET page verify the shown settings and press the "Edit Configuration ..." button.
  12. On the "General Tab" modify the IndexPath application setting to point to the repository index directory.
  13. You are finished now and should be able to browse to the search page by entering "http://yourserver/svnwebquery
