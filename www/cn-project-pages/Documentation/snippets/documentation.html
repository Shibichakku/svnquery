Summary

SvnQuery helps you finding files in your subversion repository like Google helps you finding pages on the internet. It can do a full text search for content and certain metadata like filename, path, author and so on.

How does it work?

SvnQuery maintains a full text index (powered by Lucene) of the repository which is used for fast querying. This index needs to be created once for an existing repository. When the repository changes the index needs to be updated. If you have access to the repository server you can use a post-commit hook to have your index always up to date. Indexing is done in a real detached background process, so users don't notice any delay when committing to the repository.

Alternatively you can choose to do scheduled updates, e.g. once a day. This is useful f you want to index a remote subversion server where you cannot use hooks. On Windows you can use Scheduled Tasks to do this

SvnQuery's frontend is an ASP.NET web application similar to a typical web search engine. You enter a query and milliseconds later you see your hits. You can not only search the head revision but also all revisions at once or an arbitrary custom revision range. There is one core library, SvnQuery.dll, which implements a simple but powerful query language especially suited to query code repositories. Hopefully other application, tools or addins will make use of this library.

Detailed Documentation

SvnQuery currently consists of three parts

  1. SvnIndex, a command line tool for creating and updating the full text index
  2. SvnWebQuery, an ASP.NET web application for querying the index
  3. SvnQuery.dll, a programmers library which can be used to query the index

SvnIndex - Maintaining the repository index

The command line tool SvnIndex maintains the full text index of a subversion repository. It has two subcommands for creating and updating an index.

svnindex create [Options]

Creates a new index at IndexFolder for the repository located at RepositoryUrl. Any existing content at IndexFolder will be overwritten. The optional Revision parameter specifies up to which revision the index will be created. If Revision is omitted the head revision will be used. Depending on the size of your repository indexing could take quite a long time (probably hours). It is a good idea to initially index only a few revisions and check if the basic mechanisms work. Then update the index later.

svnindex update [Options]

Updates an existing index at IndexFolder for the repository located at RepositoryUrl. The optional Revision parameter specifies the youngest revision to be included in the index. If Revision is omitted the head revision will be used. Revision needs to be greater than the index revision and smaller or equal to the repositories head revision.
