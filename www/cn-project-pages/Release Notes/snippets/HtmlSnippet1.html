1.1.4.0: 2011-03-13

  • SvnIndex: added the -b option "IgnoreBinary". This is a regex and all files that match this regex will be indexed even if their mime-type is a binary one.
  • SvnWebQuery: now uses SyntaxHighlighter 3.0
  • fixed issues #9 and #10
  • updated online documentations

1.1.3.0: 2010-11-02

  • Fixed issues with maximum number of terms per document.
  • Added support for sharing the same website for multiple indices. see http://svnquery.tigris.org/issues/show_bug.cgi?id=8
  • Added convenience logic to enable SvnFind to show file content if the repository was moved/copied and the original external url is no longer available.

1.1.2.0: 2010-08-21

  • RunDetached: detached background process is now started with a priority below normal
  • SvnWebQuery: fixed some issues in syntax highlighting
  • SvnWebQuery: allow running with under an service account that as no user profile
  • SvnIndex: Increased the number of terms per document from 10000 to 50000

1.1.1.0: 2010-07-22

  • SvnQuery: fixed resource leak in SvnApi (event handlers were not removed when putting a SvnClient back to the pool)
  • SvnQuery: fixed wrong detection of path query when querying for dot separated identifiers like "Status.OK"
  • SvnIndex: all logging is done to stdout including all error messages
  • SvnIndex: If you ommit options like -x or -n on the update command and they were set on the last call, then now those are overwritten with the default values
  • SvnWebQuery: set focus back to query input control after postback/refresh
  • SvnFind: fixed multiple UI issues
  • SvnFind: viewing application can be configured with ViewWith parameter, default is now notepad.exe
  • SvnFind: if an repository contains an index folder, you can directly open the repository as an index
  • upgraded to ReSharper 5

1.1.0.0: 2010-05-17

  • added SvnFind as a rich windows client frontend, no need to use a web server
  • workaround for unreadable files due to "inconsistent line ending" exceptions thrown by the subversion api
  • internal refactoring of the SvnQuery Api

1.0.1.0: 2010-01-13

  • SvnIndex: option -f  (filter) now defaults to "/tags/". This should prevent novice users from mistakenly indexing tags which could make the index extremely big.
  • Fix: If the external repository needs authentication SvnWebQuery didn't work. Now the credentials provided to SvnIndex with the -u and -p options are stored as index properties inside the index. This allows SvnWebQuery to actually read content from the repository and show it on the View page.

1.0.0.0:  2009-11-21

  • fine tuned syntax highlighting css styles
  • query parameters are now encoded inside the url so you can send links to live query results
  • additional testing for the first 1.0 release

0.8.0.0:  2009-10-26

  • added syntax highlighting when displaying source code from the repository

0.7.2.0:  2009-10-20

  • Fix: Queries over a range of revisions or an explicit revision could throw a NullReferenceException
  • Fix: Broken Layout on Firefox and IE7

0.7.1.0:  2009-10-19

  • SvnWebQuery: Moved from a table based layout to css based layout. Improves load time and fixes some layout problems with firefox.
  • build process uses 7zip instead of ZipFile task to prevent corrupted zip files
  • resharped the source a little with resharper 4.5

0.7.0.0:  2009-06-15

  • new option -s for generating a "single revision" index, which contains only one revision of an index (without history)
  • much more documentation about how to install and use
  • fixed handling of thread pool exceptions

0.6.0.0:  2009-04-07

  • support for subversion 1.6
  • some documentation
  • cosmetic bugfixes
