Prerequisites:

  • Windows XP or higher
  • .NET Framework 3.5

Download a self running demo of SvnWebQuery with an already created index.

  • SvnWebQuery 0.3 with an index of subversion up to revision 10000 (55MB)
  • SvnWebQuery 0.3 with an index of svnquery (1MB)

 Start SvnQueryDemo.exe to open the web ui for querying the index. Enter your query and watch your results.

Some fun queries to play with (if you have downloaded the subversion index):

  • .c => find all files ending with ".c"
  • .c a:cmpilato => find all items ending with ".c" whose auhor was "cmpilato"
  • should never happen => find all items containing the words "should never happen" in their content or path name
  • c:"should never happen" => find all items containing the exact phrase "should never happen" in their content
  • t:application* => find all binary items (svn:mime-type starting with application)

Now switch to the "All" revisions options. The search will now go through every item in every revision. Repeat the searches from above and notice the differences. Searching for revision messages works best if you select all revisions, because obiviously the head revision has only one message.

  • m:drink => find all items containing the word "drink" in their message log
  • m:(polish translation) => find all items containing the word "polish" and "translation" in their message log
